- **운영 환경**:
  - 봇 클라이언트: 안드로이드 폰 + Messenger Bot R (예정)

- **DB 출처**: 게임 파일 직접 추출 (mhdb-wilds-data 기반) + 인벤/나무위키 보강

## 2. `.정보 [몬스터]`

몬스터 사냥에 필요한 핵심 정보 제공. **카톡방 질문의 약 41% 차지 (가장 자주 호출됨).**

### 매칭 규칙
- 정확한 이름 또는 별칭 사용 (별칭 사전 참조)
- 띄어쓰기 차이는 자동 매칭 (`진 다하드` ↔ `진다하드`)

### 출력 형식
```
[이명] [한글 이름]

[약점 부위]
{부위명} : 참격 {N} / 타격 {N} / 탄 {N}
...

[약점 속성]
{속성} {★...} / ...

[상태이상]
{상태} {★...} / ...

[특수효과]
{효과} {★...} / ...

[함정·도구]
유효: {...}
무효: {...}
```

### 처리 규칙

**[약점 부위]**
- 육질 45 이상인 부위만 표시
- 그 부위에서 **45 이상인 속성만** 표시 (참격/타격/탄)
- 부위 내 최댓값 높은 순 정렬

**[약점 속성]**
- `element` (화/수/뇌/빙/용)
- 별 개수로 강도 표시 (★=1, ★★=2, ★★★=3)
- 별 개수 높은 순 정렬

**[상태이상]**
- `status` (마비/독/수면/폭파)
- 별 개수 높은 순 정렬

**[특수효과]**
- `effect` (섬광/기절/소모)
- 별 개수 높은 순 정렬

**[함정·도구]**
- `valid_items`에서 effective true/false로 분리
- 유효 / 무효 둘 다 표시

### 출력 예시 (진 다하드)
```
동봉룡 진 다하드

[약점 부위]

대동성갑각(노출 후) : 참격 75 / 타격 75 / 탄 45
동성갑각(허리) : 참격 75 / 타격 75 / 탄 45
머리 : 참격 65 / 타격 65 / 탄 45
목 : 참격 60 / 타격 60 / 탄 45
꼬리 : 참격 55 / 타격 55 / 탄 45
왼쪽 앞다리 : 참격 45 / 타격 45
왼쪽 뒷다리 : 참격 45 / 타격 45
오른쪽 뒷다리 : 참격 45 / 타격 45
오른쪽 앞다리 : 참격 45 / 타격 45

[약점 속성]
화 ★

[상태이상]
폭파 ★★ / 마비 ★ / 독 ★ / 수면 ★

[특수효과]
섬광 ★ / 기절 ★ / 소모 ★

[함정·도구]
유효: 섬광탄, 마비덫
무효: 음폭탄, 구멍함정
```

## 3. `.스킬 [스킬명]`

스킬 효과 정보 제공.

### 매칭 규칙
- **정확한 이름만** (`약점 특효`, `약점특효` ✓ / `약특`, `약점` ✗)
- 띄어쓰기 차이는 허용

### 출력 형식
```
[스킬] {스킬명} ({종류} / {카테고리})

{전체 설명}

[레벨별 효과]
Lv1: {효과 설명}
Lv2: {효과 설명}
...
```

### 종류
- `방어구` (kind=armor)
- `무기` (kind=weapon)
- `세트` (kind=set, 시리즈 효과)
- `그룹` (kind=group, 그룹 효과)

### 출력 예시 (약점 특효)
```
[스킬] 약점 특효 (방어구 / 공격)

몬스터를 공격할 때
명중한 부위가 꽤 효과적이면
회심률이 오른다.

[레벨별 효과]
Lv1: 유효 부위 공격 시 회심률+5%
     상처 공격 시 회심률+3% 추가
Lv2: 유효 부위 공격 시 회심률+10%
     상처 공격 시 회심률+5% 추가
...
```

## 4. `.스킬 [스킬명] 장비`

해당 스킬을 가진 장비 목록 표시.

### 출력 형식 (카톡 미리보기 고려, 적은 항목 먼저)
```
[스킬] {스킬명} - 보유 장비

장식주 ({개수})
- {장식주명} Lv{레벨}

호신구 ({개수})
- {호신구명} (Lv{레벨})

방어구 ({개수})
[Lv1]
- {시리즈명}: {부위들}
...
```

### 정렬 순서
장식주 → 호신구 → 방어구/무기 (적은 순서대로, 카톡 미리보기에 핵심 정보 먼저 노출)

### 시리즈/그룹 스킬도 동일한 형식 사용

## 5. `.소재 [소재명]`

소재 획득처 정보.

### 출력 형식
```
[소재] {소재명}

[획득처]
{몬스터명} ({랭크})
  - {획득방법} {확률}%
  ...
```

### 처리 규칙
- 사용처는 표시하지 않음 (획득처만)
- 같은 몬스터·랭크·방법이 여러 행인 경우 확률 **평균값**으로 합산
- **몬스터 정렬**: 해당 몬스터에서 이 소재의 최고 확률 높은 순
- **방법 정렬**: 확률 높은 순 (내림차순)
- 랭크 표기: `low` → 하위, `high` → 상위

### 출력 예시 (수호룡의 비늘)
```
[소재] 수호룡의 비늘

[획득처]
조 시아 (하위)
  - 부위 파괴 보수 100%
  - 갈무리(부패한 절단 부위) 100%
  - 갈무리 50%
  - 갈무리(부패) 50%
  - 상처 파괴 보수 50%
  - 타겟 보수 15%

수호룡 알슈베르도 (하위)
  - 부위 파괴 보수 100%
  - 갈무리(부패한 절단 부위) 100%
  - 갈무리 50%
  - 갈무리(부패) 50%
  - 상처 파괴 보수 45%
  - 타겟 보수 20%
```

## 7. `.챗 [질문]`

자유 형식 질문은 Claude API로 처리.

### 동작 흐름 (RAG 방식)
```
사용자: ".챗 마비 세팅 추천해줘"
   ↓
1. 질문에서 키워드 추출 ("마비")
2. 관련 DB 데이터 추출 (마비 잘 통하는 몬스터, 마비 무기, 마비 강화 스킬 등)
3. Claude API 호출:
   - 시스템 프롬프트: "주어진 DB 정보 안에서만 답해. 모르면 모른다고 해."
   - 사용자 질문 + DB 컨텍스트 함께 전달
4. 응답 받아서 사용자에게 전송
```

### 비용
- Claude Haiku 4.5: 약 1~2원/질문
- Claude Sonnet 4.5: 약 5~10원/질문

(추천: Haiku로 시작, 품질 부족하면 Sonnet으로 업그레이드)

### ⚠️ 미정 사항
- 어떤 모델 쓸지
- 시스템 프롬프트 구체적인 내용
- DB 컨텍스트 추출 로직 (키워드 매칭? 임베딩 검색?)

- 무기 별칭 (디씨 가이드 링크용은 이미 `external_guides.json`에 있음)
- 아이템 별칭

### 3. 검색 미스 처리
- 못 찾았을 때 응답 형식 (예: "유사 몬스터: ○○를 찾으셨나요?")

### 4. 추가 명령어 (필요 시)
- `.셋 [시리즈명]` (방어구 시리즈/그룹 효과)
- `.무기 [무기명]` (무기 스펙)
- `.속성 [속성명]` / `.상태 [상태명]` (역검색)
- `.도감` (랜덤/탐색)

### 5. `.챗` 구현 세부사항
- Claude 모델 선택 (Haiku vs Sonnet)
- 시스템 프롬프트 작성
- DB 컨텍스트 추출 로직
- 토큰 사용량 모니터링
